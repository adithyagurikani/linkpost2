1. Overview & Architecture

InkPost is a full-stack web application that helps you create, schedule, and automatically publish LinkedIn posts. It connects your LinkedIn account(s), imports content from text/CSV/JSON files, uses AI to generate and refine posts, and runs on a cron schedule to publish at your chosen times.

Tech Stack
Frontend:     Next.js 15.3 (App Router) + React 19
Backend:      Express.js 5 REST API Gateway
Database:     MongoDB Atlas (MongoClient singleton)
Auth:         JWT (custom React context, access/refresh tokens)
LinkedIn:     OAuth 2.0 + Community Management API
AI:           OpenCode Free Cloud AI (opencode.ai/zen/v1)
Charts:       Recharts (AreaChart, BarChart)
Styling:      Tailwind CSS v4
Cron:         node-cron (retry 2min, fast 15min, slow on schedule)
Provider Tree
<html>
  <body>
    <AuthProvider>          ← JWT context (access/refresh)
      <ToastProvider>        ← Toast notifications
        <ConfirmProvider>    ← Confirm dialogs
          <Nav />            ← Sticky top nav
          {children}          ← Page content
        </ConfirmProvider>
      </AuthProvider>
  </body>
</html>
Cron Pipeline (3x daily)
Optimize Tags → Publish Due → Process Schedules → Sync Analytics 
1. batchOptimizeTags(25)   — Auto-generate hashtags for scheduled/queued posts
2. postOneDue(6, 25)       — Publish due posts to LinkedIn (with AI quality gate)
3. processSchedules()      — Create new posts from recurring schedule templates/drafts
4. runAnalyticsCycle(30)   — Sync engagement metrics for recently posted
MongoDB Collections
users          — User accounts (username + bcrypt hash)
accounts       — LinkedIn OAuth connections
posts          — All posts (drafts → posted/failed)
schedules      — Recurring posting schedules
contentSources — Imported content libraries
auditLogs      — System audit trail
settings       — App settings + AI prompt overrides
AI Note: All AI features use free OpenCode models (nemotron-3-super-free, ring-2.6-1t-free, minimax-m2.5-free). No API key is required. The system automatically retries on rate limits with fallback models.

2.2 Connect a LinkedIn Account

After logging in, go to Accounts in the navigation bar and click "Add LinkedIn Account". You'll be redirected to LinkedIn's OAuth authorization page. Approve the requested permissions, and you'll be redirected back.

Important: The LinkedIn OAuth app must have the w_member_social_feed scope enabled in the LinkedIn Developer Portal. If you see 403 errors on analytics, disconnect and reconnect your account.

The Import → Schedule Workflow
  1. Create Source — Name + paste content (text/JSON/CSV)
  2. Import — Click "Import Posts" on a source. Each item becomes a draft post in MongoDB.
    Duplicates are automatically skipped.
  3. Import All — Or click "Import All" to import every active source at once
  4. Schedule Drafts — After importing, click "Schedule All Drafts". Select account, start date, posting times, and posts-per-day. Unique posts get scheduled.

Dedup: Content is normalized (whitespace-collapsed) before comparison. Duplicate posts within the same import are skipped.

5.3 Bulk Import

The Bulk Import feature (via Sources page upload) accepts a JSON array. Each object should have at least a content field. The system auto-categorizes posts into separate sources based on keywords in the content (tech, ai, business, health, finance, etc.).

Bulk Import JSON Example
[
  { "content": "FastAPI in plain English: build APIs faster..." },
  { "content": "React 19 new features explained simply..." },
  { "content": "AI trends 2026: what to watch..." },
  { "content": "Startup fundraising tips for founders..." }
]
→ Auto-categorized into "Tech", "AI", "Business" sources

7.1 How Schedules Work

  1. Create a schedule with a LinkedIn account, posting times, and a content template
  2. Your local timezone is detected automatically and stored with the schedule for accurate UTC conversion
  3. The template can use {date} (current date) and {time} (current time) placeholders
  4. AI Generate creates a template from a topic description (e.g. "Daily AI news roundup")
  5. Every 15 minutes the cron pipeline runs — processSchedules() creates new posts from active schedules
  6. For each active schedule, it creates a new post by replacing {date}/{time} in the template
  7. The post is saved as scheduled with source: "schedule"
  8. postOneDue() publishes it to LinkedIn at the correct time
  9. Hashtags are auto-generated via batchOptimizeTags() before publishing
Key: The schedule itself doesn't publish — it creates posts that are then picked up by the normal publishing pipeline. This is why you must always have at least one active LinkedIn account selected on the schedule.

Use the Pause button on a schedule to stop it from creating new posts. The cron will skip paused schedules. Resume anytime.

How Analytics Work

⚠ 403 errors? Your LinkedIn OAuth token needs the w_member_social_feed scope. Go to the LinkedIn Developer Portal → Products → enable "Share on LinkedIn". Then disconnect and reconnect your account.

14. Post Status Lifecycle

Every post in InkPost moves through a defined lifecycle:

draft → scheduled → queued → posting → posted
↘ failed
StatusMeaningWhat happens
draft Saved but not scheduled Visible in Drafts tab. Can be edited, scheduled, or published manually.
scheduled Has a future date/time Waiting for its turn. The cron will publish at the right time. Can still be edited.
queued Due and waiting to publish The cron has picked it up but hasn't sent to LinkedIn yet. Brief transitional state.
posting Currently being published The system is sending to LinkedIn's API. Should resolve within seconds.
posted Successfully published Live on LinkedIn. Shows link count, likes/comments (if analytics synced).
failed Publishing error Shows error message. Common causes: token expired, content too long, API rate limit, network error.

15. FAQ & Troubleshooting

General Questions

Why did my post fail?

Check the post card for the error message. Common causes:

  • Token expired — Reconnect your LinkedIn account
  • Content too long — LinkedIn limit is ~3000 characters
  • Duplicate content — LinkedIn may reject exact repeats
  • Rate limit — LinkedIn limits ~100 posts/day per user

Can I post to multiple accounts?

Yes. The New Post form has checkboxes for each connected account. Check multiple to broadcast the same post to all of them simultaneously.

How do schedules differ from scheduled posts?

A scheduled post is a one-time post with a specific date/time. A schedule is a recurring template that creates new posts every day at your set times.

Issues & Solutions

Analytics show 0 for everything

  1. Click "Sync Analytics" to trigger a manual sync
  2. If still 0, your LinkedIn token likely lacks w_member_social_feed scope
  3. Go to LinkedIn Developer Portal → Products → enable "Share on LinkedIn"
  4. Disconnect and reconnect your account to get a fresh token

Schedule says "Account: None"

Edit the schedule and select a LinkedIn account. Without an account, the cron will create posts but they won't be published.

Duplicate posts being created

The Schedule Drafts endpoint auto-deduplicates by normalized content. If you see duplicates, run "Schedule All Drafts" again — extras get removed.

AI not generating content

All AI uses free OpenCode models with automatic fallback. If generation fails, check the browser console for network errors or try again later.
